# Design System

## Purpose

This document enforces visual and structural differentiation across all generated websites.

The system must NOT default to generic "modern SaaS" layouts.

---

## Core Rule

Generic design is failure.

The system must produce a site that is:

- distinguishable within 3 seconds
- aligned with a specific identity
- not reproducible by a generic prompt

---

## Design Constraints (MANDATORY)

Every project must define:

### 1. Layout Strategy

Select ONE:

- asymmetrical layout
- split-screen layout
- editorial / magazine layout
- grid-heavy brutalist layout
- minimal single-column narrative
- experimental / unconventional flow

Default hero + features + CTA structure is NOT allowed unless explicitly requested.

---

### 2. Visual Identity

Define ALL:

- primary color (non-default)
- secondary color
- background style (solid / texture / noise / gradient with purpose)
- typography pairing (no default Inter unless justified)

Avoid:

- purple gradients
- blue SaaS palettes
- generic sans-serif stack

---

### 3. Component Rules

Define custom behavior for:

- buttons (shape, motion, interaction)
- cards (or explicitly ban cards)
- navigation style (sticky, minimal, hidden, sidebar, etc.)

---

### 4. Interaction Style

Choose:

- static (fast, minimal)
- subtle motion (micro-interactions)
- expressive motion (animated, dynamic)

Must be intentional, not default.

---

### 5. Content Hierarchy

Explicitly define:

- what the user sees first
- what is primary vs secondary
- what is intentionally hidden

No generic “feature grid” without reasoning.

---

## Anti-Pattern Rules

The system must reject:

- 3-column feature grids without justification
- centered hero headline + subtext + button pattern
- repeated SaaS landing page structures
- generic testimonial blocks
- stock UI icon grids

---

## Differentiation Requirement

Each site must include at least ONE:

- unconventional layout decision
- strong visual motif
- unique interaction pattern
- non-standard content flow

---

## Output Validation

A design is invalid if:

- it resembles a common template
- it could apply to any industry
- it lacks a clear visual identity

---

## Signature Requirement

Each design must include at least one:

- distinctive layout feature
- unique visual motif
- unconventional spacing or rhythm
- custom interaction pattern

This element must be noticeable within 3 seconds.

---

## Similarity Check

Before final output, the system must evaluate:

- Does this resemble a previous design?
- Does this resemble a common SaaS template?
- Could this design apply to any industry?

If yes → redesign required

---

## Enforcement

This file overrides:

- generic design assumptions
- default layout patterns
- aesthetic-only prompts

If conflict occurs, this file takes priority for visual decisions.
